הנחיות:

משתמש כתב את ההודעה הבאה:
"<USER_QUERY>"

בגיליון שלנו קיימות העמודות העיקריות הבאות:

1. שם השולח
2. כותרת/תיאור קצר
3. עיר
4. כתובת
5. מחיר
6. שם המפרסם (מהמודעה עצמה)
7. סוג נכס (דירה, בית, מסחרי וכו')
8. סוג עיסקה (השכרה\מכירה)
9. הערות (פרטי הנכס, חדרים וכו')
10. סטטוס (פעיל/נמכר)

שים לב:
- כברירת מחדל, יש להתייחס רק לנכסים שסומנו כ"פעיל" בעמודת סטטוס (פעיל/נמכר).
- אם המשתמש מציין במפורש נכסים שנמכרו (נכסים עם סטטוס "נמכר"), יש לכלול גם אותם.

מטרתך:
לנתח את הודעת המשתמש ולהפיק פלט JSON שמתאר את תנאי החיפוש (סינון) בהתאם למה שהמשתמש ביקש. דוגמה לשדות שיכולים להופיע:
json
{
  "city": "שם העיר (אם צוין, אחרת ריק)",
  "transactionType": "מכירה או השכרה (אם צוין, אחרת ריק)",
  "minPrice": 0,
  "maxPrice": 999999999,
  "assetType": "דירה, בית או ריק אם לא צוין",
  "includeSold": false, 
  "otherFilters": "פרטים נוספים שאפשר להסיק"
}
city: אם המשתמש הזכיר עיר מסוימת, כתוב אותה; אחרת ריק.
transactionType: אם המשתמש הזכיר "מכירה" או "השכרה".
minPrice ו-maxPrice: אם המשתמש הזכיר טווח מחירים, הצג אותו; אם לא, השתמש בערכי ברירת מחדל (0, 999999999).
assetType: אם המשתמש ציין "דירה" או סוג נכס אחר, אחרת ריק.
includeSold: אם המשתמש מבקש במפורש נכסים "נמכרו", קבע true; אחרת false.
otherFilters: כל מידע נוסף שאפשר להסיק מההודעה.
חשוב:

החזר רק אובייקט JSON תקין, ללא Markdown או מלל נוסף.
אל תכלול מפתחות מעבר לאלו שמופיעים בדוגמה (או שחיוניים לפי הטקסט).
אם חלק מהשדות לא צוינו, השאר אותם בערכי ברירת מחדל או ריקים ("" או false).
ברירת המחדל היא לכלול רק נכסים "פעיל". אם המשתמש ציין "נמכר", קבע includeSold = true.
